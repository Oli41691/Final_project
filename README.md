## Автоматизация тестирования интернет-магазина "Читай-город" 
# Описание проекта
Данный проект предназначен для автоматизированного тестирования сайта интернет-магазина "Читай-город". В рамках проекта реализованы UI и API тесты, что обеспечивает контроль качества и стабильности сайта за счет автоматической проверки основных сценариев работы. Используются инструменты Selenium WebDriver, pytest и Allure для формирования отчетов.

## Необходимые зависимости
# Установите все необходимые библиотеки командой:
pip install -r requirements.txt

В requirements.txt обычно указаны:
selenium (для UI тестирования)
pytest (для запуска тестов)
requests (для API)
allure-pytest и allure-python-commons (для отчетов)


## Инструкции по запуску тестов
1. Запуск UI тестов
pytest -m ui

2. Запуск API тестов
pytest -m api

3. Запуск всех тестов
pytest

4. Формирование отчёта Allure
Запускаете тесты с сохранением результатов:
pytest --alluredir=allure-results

Генерируете и просматриваете отчет:
allure serve allure-results

# Работа с конфиденциальными данными
- В файле config.py рекомендуется хранить токены, ключи API и иные секретные параметры.
- Для безопасной работы используйте переменные окружения или .env файлы.
- Не забывайте исключать секретные данные из коммитов (.gitignore).

# Руководство по ручному тестированию
## Описание проекта:

Проект предназначен для проверки функциональности интернет-магазина "Читай-город", который позволяет искать и приобретать книги онлайн, добавлять товары в корзину и оформлять заказы. Руководство включает сценарии ручного тестирования, проверки работоспособности ключевых сценариев.


# Дополнительная информация
- Перед запуском убедитесь, что драйвер браузера (например, ChromeDriver) установлен и настроен.
- В конфигурационных файлах можно указать параметры запуска, такие как URL сайта, время ожидания и т.п.
- Для формирования Allure-отчета необходимо установить сам allure-commandline:
pip install allure-python-commons
